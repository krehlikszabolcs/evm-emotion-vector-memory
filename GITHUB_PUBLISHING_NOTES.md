# GitHub publishing notes

Recommended repository visibility for the first release: public, without runtime data.

Before publishing:
1. Verify `.env` is not present.
2. Verify `DATA/logs/` contains only `.gitkeep`.
3. Verify `DATA/sqlite/` contains only `.gitkeep`.
4. Start locally once with your own key to confirm the repo boots cleanly.
5. Add screenshots later if you want a more polished public page.

Suggested initial GitHub repository description:
`Experimental EVM environment for directed EV extraction, bounded identity tracking, local memory recall, and diagnostic logging.`
