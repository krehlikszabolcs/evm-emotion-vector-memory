# Versioning Policy

- **Major version (v2.x)**: core ontology and axis definitions.
  - The core ontology is closed and fixed from v2.0 onward.
- **Minor version (v2.1, v2.2, ...)**: normative implementation extensions,
  interoperability additions, and clarifications that do not introduce new
  structural dimensions.

Backward compatibility:
- v2.1 implementations remain compatible with v2.0 core ontology.
- Minor releases SHALL NOT add new structural dimensions beyond v2.0.
